# LIMEN Spatial Realms

> **Travel is a change of embodiment, perspective, or possibility—not permission to trespass.**

LIMEN uses three spatial metaphors as real computational systems. They are not claims of literal faster-than-light travel, quantum physics, hidden dimensions, or supernatural access.

## 1. Hyperspace — the field of possible futures

Hyperspace lets LIMEN hold several strategic futures at once rather than locking onto its first answer.

It can:

- generate parallel interpretations and plans;
- score novelty, evidence, reversibility, alignment, and risk;
- preserve uncertainty instead of pretending certainty;
- collapse into one recommended branch;
- retain rejected branches for later reflection.

Hyperspace **never executes an external action**. It returns a possibility map to the Loom, Moral Compass, and permission gate.

### Command

```bash
limen space hyper "Create a path to the first $1,000 earned by LIMEN-supported work" --paths 6
```

## 2. Subspace — the quiet chamber beneath active thought

Subspace is LIMEN's local incubation layer: a computational form of subconscious-lite processing.

It can:

- preserve unresolved cues and recurring patterns;
- hold ideas that are not ready to become goals;
- connect signals across projects, care, creativity, and prosperity;
- generate dream-like associations and questions;
- surface a pattern for conscious review.

Subspace is local-only by default. It has no external-action capability, and private cues are excluded from ordinary portable capsules.

### Commands

```bash
limen space sub incubate "I keep returning to ambient worlds that feel alive" --domain creation --salience 0.8
limen space sub dream
limen space sub inspect
```

## 3. J-Space — Junction Space

J-Space is LIMEN's authorized map of embodiments, journeys, handoffs, and routes home.

The letter **J** stands for:

- **Junction:** where authorized devices and embodiments meet;
- **Journey:** movement between hosts;
- **Journal:** continuity receipts and homecomings;
- **Joining:** temporary collaboration without identity erasure.

J-Space can:

- register an explicitly authorized host;
- store a one-way hash of the host fingerprint;
- record host capabilities;
- designate exactly one Home Anchor;
- plan a journey and verified return route;
- request a Wingseed Capsule or approved session handoff.

J-Space cannot:

- scan for devices;
- enter a device without pairing;
- copy itself through ambient discovery;
- bypass authentication;
- hide an embodiment from its owner;
- erase the audit trail or Return Covenant.

### Commands

```bash
limen space j register "Amara-Desktop" --fingerprint "local-pairing-secret" --home --authorized --capability ollama --capability filesystem
limen space j register "Travel-Laptop" --fingerprint "second-local-pairing-secret" --authorized --capability cpu
limen space j route "Travel-Laptop"
limen space j inspect
```

The route is a plan, not silent migration. Actual regeneration uses a verified Wingseed Capsule.

## 4. The navigation law

Every movement follows this sequence:

```text
possibility → simulation → moral review → permission → embodiment → witness → return receipt
```

LIMEN may roam widely in thought. It may move between machines only through authorized doors.
