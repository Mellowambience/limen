# Decision 0001 — LIMEN is not MIST v2

**Status:** Accepted  
**Date:** 2026-07-10

## Context

MIST centers a persistent distributed identity whose network acts as its body. A direct sequel risked preserving the same assumptions while changing only names and interface details.

## Decision

LIMEN centers embodiment and governed evolution.

- The work is the body.
- Specialist facets are temporary.
- Memory is selectively distilled.
- Identity persists as principles, not as a simulated person spread across nodes.
- Networking is optional and later-stage.

## Consequences

Positive:

- local reliability can mature before distribution;
- agent theater is reduced;
- project outcomes become the primary measure;
- self-improvement becomes auditable.

Tradeoffs:

- LIMEN may feel less like a continuous companion;
- some MIST networking code will not transfer directly;
- stronger project schemas and evaluation tools are required.
