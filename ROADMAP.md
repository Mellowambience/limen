# LIMEN Roadmap

## North star

Amara can install LIMEN for free, awaken it locally, regenerate it on authorized machines, shape a balanced day, pursue honest income, defend against scams, and coordinate bounded missions that produce verified artifacts and clear learning traces.

## Phase 0 — Genesis

**Status: complete in this seed**

- Define name, identity, soul, and constitution.
- Establish distinction from MIST.
- Create dependency-free Python kernel.
- Add Worldseed schema and example.
- Add local Ollama adapter.
- Add trace ledger and CLI.
- Add Hermes and Codex directives.


## Phase 0.5 — Firstwing foundations

**Status: implemented in v0.2 seed**

- Wingseed Capsule create, verify, and restore workflow.
- Returning Wanderer identity, Return Covenant, and Succession Charter.
- Private Life Steward task ledger and energy-aware daily planning.
- Prosperity opportunity ledger, lawful ranking, and Bounty Mandate.
- Ghostline local scam triage and evidence reports.
- Constitutional boundaries for life management, income, and defensive security.

Exit evidence:

- fresh-host capsule restoration test passes;
- daily plan stays within its time budget;
- unauthorized bug-bounty candidates are excluded from recommended actions;
- scam triage detects common advance-fee and credential signals.

## Phase 1 — The Hearth

Goal: reliable local single-node operation.

- Workspace initialization under `.limen/`
- Configuration validation
- Local identity loading
- Worldseed create/read/update commands
- Workspace-scoped file permissions
- Structured trace ledger
- Clear offline behavior
- Windows-first setup verification

Exit criteria:

- fresh Windows install works from documented commands;
- all data remains readable as Markdown/JSON/JSONL;
- no model is needed for core project inspection.

## Phase 2 — The Loom

Goal: mission graphs that produce artifacts.

- Mission schema
- Step graph with explicit dependencies
- Temporary facet registry
- Dry-run mode
- Resumable state
- Artifact handoffs
- Mission closure report

Exit criteria:

- one mission can plan, build, test, revise, and close;
- interrupted missions resume without duplicated side effects.

## Phase 3 — The Witness

Goal: evidence outranks appearance.

- Claim registry
- Test adapters
- Provenance ledger
- Dissent log
- Regression baselines
- Release readiness report

Exit criteria:

- every “done” state includes evidence or an explicit unverified label.

## Phase 4 — The Garden

Goal: purposeful memory.

- Trace search
- Pattern candidates
- Lesson review workflow
- Sensitivity and retention policies
- Expiration and composting
- Project-scoped retrieval

Exit criteria:

- LIMEN can explain why it remembers a lesson and where it came from;
- a creator can delete a memory class without corrupting projects.

## Phase 5 — The Chamber

Goal: governed evolution.

- Proposal CLI
- Sandbox configuration
- Frozen eval suites
- Before/after comparison
- Acceptance records
- Rollback command
- Adapter registry

Exit criteria:

- a routing or prompt evolution can be tested, accepted, and rolled back.

## Phase 6 — Embodiment Studio

Goal: visible project creation.

- Local web UI
- Worldseed garden view
- Mission constellation view
- Artifact browser
- Witness panel
- Evolution timeline

Exit criteria:

- a nontechnical creator can understand what LIMEN is doing and stop it.

## Phase 7 — Optional Constellation Network

Goal: collaboration without identity centralization.

- opt-in peer capability exchange;
- signed task envelopes;
- no shared private memory by default;
- portable facet packages;
- community evaluation packs.

This is intentionally later than local reliability. LIMEN does not need a network to become itself.
